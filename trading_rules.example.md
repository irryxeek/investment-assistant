# 交易规则

本文件由 /driven 分析时强制校验。规则分三档，不同档位有不同的灵活度。
请根据个人风险偏好和持仓情况修改各项阈值。

---

## 🔴 铁律（Iron Rules）

**绝对不允许在分析中突破**。这些规则防止灾难性错误，违反 = 系统级风险。

### 组合级强制刹车
<!-- 这些是组合层面的硬约束，建议保留但可调阈值 -->
- **集中度硬上限**：单一标的占比 > 50% → 强制减仓优先级最高
- **单日跌幅冷静期**：组合单日跌幅 < -3% → 次日禁止任何操作
- **现金极限保护**：现金占比 < 8% → 禁止任何加仓，直到现金回升至 12%

### 频率天花板
- 全组合：每周最多 2 笔（同日轮动"减A+加B"算 1 笔）
- T+1 未确认期间：不叠加同一标的的新操作

> 铁律不接受任何理由的突破。建议条目少而硬，只放真正的"灾难防火墙"。

## 🟡 默认规则（Default Rules）

**可以突破，但需满足三个条件：**
1. 在操作建议中**显式标注 "⚠️ 突破默认规则: [规则名]"**
2. 给出**具体的市场环境理由**（不能是"我觉得应该卖"）
3. 操作后在 `trade_history.md` 中**记录这次突破**及事后效果

### 频率与冷静期
- 同一标的：每周最多操作 1 次
- **反向操作冷静期**：减仓后 5 个交易日内禁止加仓；加仓后 5 个交易日内禁止减仓
  - 目的：防止追涨杀跌、短期波动驱动的来回操作

### 多维度确认
每次操作必须**至少 2 个维度同时确认**，单维度信号不足以触发：
- **减仓**：技术面 + 情绪/基本面之一（PE 高估 / 资金流出 / 波动率亢奋）
- **加仓**：技术面 + 情绪/基本面之一（资金流入 / PE 低估 / 波动率恐慌）
- 仅技术面触发 → 降级为"观察候选"，不直接执行

### 超跌反弹期禁令
- RSI 从 < 35 回升过程中：**仅禁止加仓**，等 RSI 站稳 50 以上再评估加仓
- 反弹日减仓不受此禁令约束

### 现金缓冲规则
- 现金占比 10-15%：只允许减仓，不允许加仓

## 🟢 软参考（Soft Reference）

**纯参考，无需特殊标注**。用于辅助判断方向。

### 市场温度调节
根据"盘面温度"动态调整心理倾向（不强制改变规则数值）：
- **恐慌期**（恒生波指 > 25 / VIX > 30）：倾向加仓，减仓需更多确认
- **亢奋期**（恒生波指 < 15 / VIX < 12）：倾向减仓，加仓需更多确认
- **中性期**：按默认规则执行

### 消息面权重
- 地缘事件、政策动态：影响操作时机，但不改变硬规则阈值
- 重大事件前 24 小时：倾向不操作，等结果落地
- 资金面：作为加仓的辅助确认，不单独触发操作

### 仓位目标（中期，非每日校验）
<!-- 根据你的资产配置目标修改，"当前"列在更新持仓后手动同步 -->
| 维度 | 目标 | 当前 |
|------|------|------|
| 权益类占比 | 自定义 | 待填写 |
| 现金 | 15-20% | 待填写 |
| 避险资产 | 8-12% | 待填写 |

### 估值参考
- PE 历史分位 < 20%：倾向加仓
- PE 历史分位 > 80%：倾向减仓
- 仅作为方向性参考，不覆盖默认规则

## /driven 输出规范

### 日常简报（无规则触发）
≤ 5 行：
```
规则校验：无触发。[简述原因]
盘面温度：[恐慌/谨慎/中性/乐观/亢奋]，[一句话宏观背景]
评分变化：[标的A 持有→持有]
关注事项：[最重要的1条]
```

### 完整分析（默认规则触发或重大事件时）
1. **核心结论**
2. **大盘复盘**（指数/波动率/资金/油价/汇率 → 盘面温度）
3. **持仓个股分析**（每个标的的四维分析）
4. **五级评分表**
5. **操作建议**（必须包含：3 风险点 + 2-3 催化 + 关键位 + 多空辩论 + 具体金额 + 多维度确认 + 操作的风险 + 不操作的代价）
6. **下一步关注**

### 突破默认规则的特殊格式
```
⚠️ 突破默认规则：[规则名]
原规则要求：[原始阈值]
当前实际值：[偏离值]
突破理由：[具体市场环境，不少于 2 句]
事后追踪计划：[操作后 N 天内观察的指标]
```

## 规则版本控制

- 修改铁律：需要用户明确同意，且需在更新记录中说明
- 修改默认规则：需要用户明确同意
- 修改软参考：可在分析中讨论调整方向
- **任何规则修改都不能在当次分析中生效**——防止"为操作找理由"
